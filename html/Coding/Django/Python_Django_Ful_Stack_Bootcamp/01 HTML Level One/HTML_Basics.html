HTML Basics Chapter 1:

Summary:

The Front End of websites are mostly comprised of 3 parts:

  1. HTML: Hyper Text Markup Language is a scripting language, and the main structure and contents of the page. This defines what text goes where, hyperlinks, images, tables, etc.
  2. CSS: Cascading Style Sheets defines the formatting and style of the HTML contents. This is what will define fonts, colors, etc.
  3. JavaScript: JavaScript allows for interactive elements of the website with the Back End. This is what allows for uploads, downloads, searches, etc.

The Back End of websites are also mostly comprised of 3 parts:

  1. Programming Language: Coding languages such as Python or Rust.
  2. Frameworks: Connect the Development language to the front end. Django is what we'll use.
  3. Databases: Store all the data in the background, and are accessed from the coding language.

1.1: Headings, Ordered Lists, Unordered Lists, Bold, and Italics

  1. Headings are added by using <h1> </h1> for h values from 1 to 6.
  2. Ordered lists can be made by using <ol> </ol>.
  3. Unordered lists can be made by using <ul> </ul>.
  4. List items on an ordered or unordered list are added with <li> </li>.
    • Lists may also be nested by simply nesting an ordered or unordered list within a list.
  5. Font is made Bold with <b> </b> or made Italic with <i> </i>
  6. Lorem Ipsum text can be inserted with simply typing "lorem"

Lastly, simple paragraph text can be added with <p> </p>.

1.2: Divs and Spans

Div tags can be added with <div> </div> and would have a specific class defined in them as seen below. These div tags can specify certain attributes to the class indicated to later define things like font and color.

For example:

<div class="groupone1">

Lorem ipsum, dolor sit amet consectetur adipisicing elit. Esse quisquam hic illum unde enim id repellendus, voluptate doloribus adipisci ea dolore consequatur, consectetur neque nisi vel non sequi tenetur? Magni!

</div>

Span tags however, can be added to specific text in the middle of a paragraph using <span> </span>.

1.3: HTML Attributes

Attributes can be added to allow us to add more information to an HTML tag, such as a link to another website, or even an image.

Images should be in the same directory as a script. To insert an image, use the format:

<img src="django.png" alt="">

  • Note: This is a type of tag classified as "Self-closing" as it does not require a respective </> at the end.

HTML Basics Chapter 2:

Summary:

This Section will consist of Tables, Forms, Labels, and Selections.

2.1: Tables

Tables can be inserted with a <table border=""> tag.

The border thickness can be specified from this tag or left blank for no border.

There are then 4 main elements within a table:

  • <thead> - indicates the headers of a table, under which all <th> tags go.
  • <th> - indicates header items, such as Num, State, Capital.
  • <tr> - indicates a new Table row, under which all <td> tags go.
  • <td> - indicates a table cell, which is placed under a specific <tr> tag row.
  • <tfoot> - indicates the footer of a table which can appear above the table on top of the header by default.

2.2: Forms

Forms can be used to have a user input information such as an Email and username as shown below.

To write a form, use the <form></form> decorator separated by lines with <input> decorators in between.

An input can be written as <input type="" name="" value"">

Inputs have 3 values to associate:

  1. type: indicates the field type such as email, password, or submit.
  2. name: indicates the name of the value entered by the user.
  3. value: indicates the default value of that field before user input.

Here is an Example Login Prompt with three different inputs using the email type, password type, and submit type:

2.3: Forms with Labels

Forms can also have actions (such as a URL) to indicate something to do with the information input, and methods (such as "get") to describe what is doing with the input information.

Labels can be added to a form by wrapping a form inside of a label as shown below. by indicating for="" in your label, you can specify a specific label to a specific form.

Values can be put into forms but will need deleted. Otherwise, placeholders can be put into the field and will vanish when clicked. Below, we see placeholders for both the Email and Password fields.

Lastly, you may make a field required by adding the word required at the end of an input tag.

2.4: Forms and Selections

Radio buttons are another great type of Input, and if the inputs share the same name on multiple radio buttons, you can link them together to only allow one choice. An input should also have an id to indicate what selection was made.

A drop-down menu can be put into a form with the select tag wrapped around multiple option tags.

Lastly, you can provide a multiple-line text box instead of a single line textbox by with:

<textarea name="" id="" cols="" rows=""></textarea>

This will allow you to create a textbox that has a specified number of columns and rows in size, and a specified id just like the other inputs.
